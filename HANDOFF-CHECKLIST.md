# Handoff Checklist

- [ ] Final normalized source file is saved as `data/intake/current-program.json` or archived to the agreed path.
- [ ] `reports/validation-report.md` reflects the approved source file.
- [ ] `reports/build-report.md` reflects the approved source file.
- [ ] `reports/releases/latest-approved.json` points to the current approved release.
- [ ] `dist/current-delivery-manifest.md` reflects the current approved delivery package.
- [ ] `dist/delivery-package/` contains the files intended for final handoff.
- [ ] Any superseded release was moved into `data/archive/releases/<release-id>/`.
- [ ] Published URL is recorded and shared with the team.
- [ ] Organizer approval timestamp is recorded.
- [ ] Any last-minute changes were revalidated and rebuilt.
- [ ] Internal ops pages were not included in the client-facing handoff.
